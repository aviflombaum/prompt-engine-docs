# Configuration

PromptEngine offers flexible configuration options through Rails credentials, initializers, and
environment variables. This guide covers all configuration options to customize PromptEngine for
your needs.

## API Provider Credentials

<Callout type="warning">
  API credentials are required for the playground feature. Without them, you can still manage
  prompts but won't be able to test them.
</Callout>

### Setting up credentials

<Tabs items={["Rails Credentials", "Environment Variables"]}>
  <Tab value="Rails Credentials">
    The recommended approach is using Rails encrypted credentials: ```bash rails credentials:edit
    ``` Add your API keys: ```yaml # OpenAI Configuration openai: api_key: sk-your-openai-key
    organization_id: org-optional-org-id # Optional # Anthropic Configuration anthropic: api_key:
    sk-ant-your-anthropic-key ```
    <Callout type="info">
      You only need to configure the providers you plan to use. Having both configured gives you
      more flexibility in the playground.
    </Callout>
  </Tab>

  <Tab value="Environment Variables">
    Alternatively, use environment variables: ```bash # OpenAI export
    OPENAI_API_KEY="sk-your-openai-key" export OPENAI_ORGANIZATION_ID="org-optional-org-id" #
    Anthropic export ANTHROPIC_API_KEY="sk-ant-your-anthropic-key" ``` Add these to your `.env` file
    in development or set them in your production environment.
  </Tab>
</Tabs>

## Initializer Configuration

For advanced configuration, create an initializer:

```ruby
# config/initializers/prompt_engine.rb
PromptEngine.configure do |config|
  # Default AI model for playground
  config.default_model = "gpt-4"

  # Feature toggles
  config.enable_playground = true
  config.enable_versioning = true
  config.enable_parameter_validation = true

  # Authentication and authorization
  config.require_authentication = true
  config.admin_role = :admin
  config.auth_method = :devise  # or :custom

  # Limits and defaults
  config.max_prompt_length = 10_000
  config.max_versions_per_prompt = 100
  config.default_temperature = 0.7

  # Caching
  config.cache_responses = true
  config.cache_ttl = 1.hour
end
```

## Database Requirements

PromptEngine is designed to work with standard Rails databases:

<Tabs items={['PostgreSQL', 'MySQL', 'SQLite']}>
  <Tab value="PostgreSQL">
    **Recommended for production**
    
    Requirements:
    - PostgreSQL 12.0+
    - JSON column support (built-in)
    - Full-text search (optional, improves search)
    
    ```yaml
    # config/database.yml
    production:
      adapter: postgresql
      encoding: unicode
      pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
    ```
  </Tab>
  
  <Tab value="MySQL">
    **Good alternative for production**
    
    Requirements:
    - MySQL 5.7+ or MariaDB 10.2+
    - JSON column support
    
    ```yaml
    # config/database.yml
    production:
      adapter: mysql2
      encoding: utf8mb4
      pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
    ```
  </Tab>
  
  <Tab value="SQLite">
    **Development only**
    
    <Callout type="warning">
      SQLite is only recommended for development. Use PostgreSQL or MySQL in production.
    </Callout>
    
    Requirements:
    - SQLite 3.38.0+
    - JSON1 extension
  </Tab>
</Tabs>

## Security Considerations

<Callout type="error">Never expose your API keys in client-side code or version control!</Callout>

### Best Practices

1. **Use Rails credentials** for API keys in production
2. **Enable authentication** if your admin routes are publicly accessible
3. **Restrict access** to the PromptEngine routes in production
4. **Rotate API keys** regularly
5. **Monitor usage** to detect any unusual activity

### Adding Authentication

<Steps>
  <Step>
    ### Configure authentication requirement
    
    ```ruby
    # config/initializers/prompt_engine.rb
    PromptEngine.configure do |config|
      config.require_authentication = true
      config.auth_method = :devise
    end
    ```
  </Step>
  
  <Step>
    ### Protect routes
    
    ```ruby
    # config/routes.rb
    authenticate :user, lambda { |u| u.admin? } do
      mount PromptEngine::Engine => "/prompt_engine"
    end
    ```
  </Step>
  
  <Step>
    ### Customize authentication
    
    ```ruby
    # app/controllers/application_controller.rb
    class ApplicationController < ActionController::Base
      def authenticate_prompt_engine_admin!
        unless current_user&.can_manage_prompts?
          redirect_to root_path, alert: "Not authorized"
        end
      end
    end
    ```
  </Step>
</Steps>
